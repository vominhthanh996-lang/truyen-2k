# Vietnamese Wasteland Webnovel

## Core Rule

Write original prose. Learn craft patterns from references, but never copy distinctive sentences, scenes, character names, or protected expression from source stories.

The target voice is Vietnamese long-form webnovel: immersive, emotional, clear, addictive, and human. Prefer lived details, hard choices, team bonds, survival economics, and chapter hooks over summary-like event coverage.

## Default Workflow

1. **Load project memory first**: read the current story bible, outline, and recent chapters before writing.
2. **Identify the task**:
   - New chapter/part: use the Chapter Draft Workflow.
   - Improve existing draft: use the Revision Workflow.
   - Learn from YouTube/transcript/reference: use the Reference Learning Workflow.
   - Build local-AI writing data: use the Dataset Workflow.
3. **Write slowly enough for quality**: avoid drafting many parts at once unless the user explicitly asks for broad rough draft. For quality prose, prefer 1-3 chapters per pass.
4. **Preserve Vietnamese diacritics**. Do not use automatic accent-restoration tools unless the user explicitly asks.
5. **End with a hook** unless writing a quiet emotional close.

## Chapter Draft Workflow

Before drafting, decide these five items:

- **Scene pressure**: What danger, need, debt, hunger, illness, rule, or emotional conflict makes this chapter necessary?
- **Human wound**: Who is hurt, ashamed, afraid, tempted, lonely, or trying to stay kind?
- **Survival texture**: What concrete item matters: water, salt, medicine, dirty cloth, crystal, tool, door, map, ration, filter, seed, shoe, debt?
- **Choice**: What does the lead choose that costs something?
- **Hook**: What changes by the last 1-3 paragraphs?

Draft with scene first, explanation second. Avoid starting chapters with abstract lore if a smell, sound, object, or human action can carry the reader in.

## Revision Workflow

When improving a draft, do not only add adjectives. Revise in this order:

1. **Slow the important moment**: add gesture, silence, physical discomfort, and the thought the character does not say.
2. **Cut summary bridges**: replace "sau đó họ..." with a scene when the moment affects relationship, danger, or status.
3. **Make dialogue carry power**: each line should reveal fear, tactic, affection, shame, debt, or hierarchy.
4. **Add consequence**: after every rescue, gift, fight, or public speech, show who pays.
5. **Sharpen the final hook**: end with a new problem, a changed relationship, or a quiet line that hurts.

## Reference Learning Workflow

Use when the user gives a YouTube link, transcript, or sample story.

1. Extract or read only enough reference material to analyze craft. If current info requires internet or a page transcript, browse or ask the user for transcript when needed.
2. Summarize craft patterns, not plot for copying:
   - chapter opening style
   - paragraph rhythm
   - emotional technique
   - escalation structure
   - "sảng điểm" placement
   - hook style
   - dialogue habits
   - worldbuilding density
3. Save reusable lessons into `references/learned-patterns.md` when the user wants long-term improvement.
4. Convert lessons into original rules for this story.
5. Write a short sample chapter or rewrite sample to test the new rule.

Never imitate a living author's exact wording or distinctive scene construction. Blend lessons into the existing story voice.

## Dataset Workflow For Local AI

Use when the user wants a local model or specialized writing dataset.

Create project-local training material, not copied copyrighted prose:

- `style-bible.md`: desired voice, themes, do/don't rules.
- `chapter-template.md`: reusable chapter structure.
- `revision-checklist.md`: quality criteria.
- `prompt-pack.md`: prompts for outlining, drafting, revising, dialogue, hooks.
- `original-examples/`: only original passages written for the user's project.

For reference stories, store analysis notes and craft labels, not raw copyrighted transcripts unless the user owns or provides permission.

## Quality Bar

A chapter is not ready if:

- It mostly summarizes events instead of dramatizing scenes.
- The protagonist wins without cost.
- Supporting characters only praise the lead.
- Food/water/radiation economics disappear.
- Dialogue could be spoken by any character.
- A rescue has no consequence.
- The ending does not make the reader want the next chapter.

Use `references/style-bible.md` and `references/revision-checklist.md` for deeper guidance when doing serious drafting or revision.

Use `references/plot-engine.md` when planning long arcs, creating chapter events, making 10-hour episode outlines, or fixing a draft that feels rushed, flat, or summary-like.
